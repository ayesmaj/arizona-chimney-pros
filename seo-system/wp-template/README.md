# Arizona Chimney Pros — WordPress Template Files

Theme-agnostic rendering layer for the programmatic SEO pages generated by `seo-system/generate.py`.

These files turn the CSV output into actual live, indexable pages with:
- Flat URLs (`/gas-fireplace-repair-phoenix/` — not `/service_area_page/gas-fireplace-repair-phoenix/`)
- JSON-LD schema injection in `<head>`
- Meta title / description / canonical overrides
- Semantic HTML with inline FAQ microdata (belt-and-suspenders with JSON-LD)
- Minimal default styling (easy to override via child theme)

## Files

| File | Purpose |
|---|---|
| `single-service_area_page.php` | Main template. Drop into active theme root. |
| `functions-snippet.php` | CPT registration + schema/meta hooks. Paste into `functions.php` OR load as mu-plugin. |

## Installation (3 minutes)

### 1. Copy the template file into your theme

```
wp-content/themes/YOUR-ACTIVE-THEME/single-service_area_page.php
```

WordPress's template hierarchy picks this up automatically for any post of `post_type=service_area_page`. No theme modification needed.

### 2. Wire up the functions snippet

Two options:

**Option A — Paste into theme's functions.php:**
Open your active (child) theme's `functions.php` and paste the *entire contents* of `functions-snippet.php` at the bottom. Do NOT include the opening `<?php` tag if functions.php already has one.

**Option B (cleaner) — Drop as a mu-plugin:**
```
wp-content/mu-plugins/acp-seo-pages.php
```
Must-use plugins load automatically and survive theme switches. Recommended for production.

### 3. Ensure ACF field group is imported

From `seo-system/acf-fields.json`:
1. WP Admin → Custom Fields → Field Groups → Tools
2. "Import Field Groups" → select `acf-fields.json`
3. Click Import

### 4. Flush rewrite rules

- Settings → Permalinks → click "Save Changes" (no changes needed — just flushes the rules).
- Without this, the flat URLs will 404.

### 5. Import pages via WP All Import

1. WP All Import → New Import → Upload `pages-generated.csv` (output of `generate.py`)
2. Target: Service Area Pages (the CPT you just registered)
3. Map CSV columns to the ACF fields:
   - `slug` → Post Slug
   - `title` → Post Title
   - All other ACF fields auto-map by field name (ACF Add-on for WP All Import handles this)
4. Run import

## CallRail Dynamic Number Swapping

When CallRail is set up, override the phone constants in `functions-snippet.php`:

```php
define( 'ACP_PHONE', '(602) 555-TRACKED' );
define( 'ACP_PHONE_TEL', '+16025551234' );
```

CallRail's JS snippet will still swap these client-side per campaign source — the constants just set the SSR default.

## Customizing the Design

The template outputs stable CSS class names under the `acp-*` prefix. To restyle:

1. Remove or neutralize the inline styles in `functions-snippet.php` (find `acp_inline_styles`).
2. Add your own CSS in your child theme's `style.css` targeting the `acp-*` classes.

Classes you can style:
- `.acp-hero`, `.acp-hero__title`, `.acp-hero__subtitle`, `.acp-hero__cta`
- `.acp-btn`, `.acp-btn--primary`, `.acp-btn--secondary`, `.acp-btn--lg`
- `.acp-section`, `.acp-intro`, `.acp-local`, `.acp-signs`, `.acp-pricing`, `.acp-process`, `.acp-trust`, `.acp-cta`, `.acp-related`
- `.acp-faq`, `.acp-faq__list`, `.acp-faq__item`, `.acp-faq__q`, `.acp-faq__a`
- `.acp-pricing__range`
- `.acp-related__list`

## Schema Verification

After importing a page, verify the JSON-LD is rendering correctly:

1. Visit the page in a browser, view page source
2. Look for `<script type="application/ld+json">` in `<head>`
3. Copy the JSON content into [Google's Rich Results Test](https://search.google.com/test/rich-results)
4. Should show: LocalBusiness, Service, FAQPage, BreadcrumbList, WebPage — all valid

## Troubleshooting

- **Pages 404:** Flush permalinks (step 4 above).
- **Schema missing:** Verify `schema_json` ACF field has content; check `View Source` on the rendered page.
- **FAQ rich results not appearing:** Google can take days/weeks. Use Rich Results Test to confirm eligibility.
- **Styles conflict with theme:** Remove `acp_inline_styles` hook, apply your own CSS.
- **URLs conflict with existing pages/posts:** The flat-rewrite fallback is additive — existing pages/posts take priority.
